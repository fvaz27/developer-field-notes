Git vs Github

Git is a version control tool that runs locally on your computer. It tracks changes to your files, lets you create snapshots called commitsm and allows you to work across branches (all without internet connection)

GitHub is a cloud-based platform that hosts Git repositories online. It lets you back up your work, work with others, open pull requests, and review code. GitHub uses Git but Git can exist without GitHub

Git is a tool. GitHub is the hosting service

The Daily Loop

Every working session follows the same four steps:

  1. Edit - make changes to your files in the working directory.
  2. Stage - use git add to move changes to the staging area.
  3. Commit - git commit to save a snapshot to your local repository.
  4. Push - use git push to send you commits to the remote on GitHub.

Git Command Reference

  • git init - initialized a new Git repository in the current folder.
  • git status - shows which files are modified, staged or untracked.
  • git add . or git add [filename(s)] - stages changes so they will be included in the next commit. Use . to stage everything at once, or specify a filename to stage one file at a time.
  • git commit -m "message" - saves a snapshot of the staged changes with a descriptive message.
  • git push - uploads local commits to the remote repositry on Github
  • git pull - fetches and merges the latest changes from the remote into your current branch.
  • git checkout -b [branch name] or the newer way git switch -c (branch name)- it creates a new branch and switches to it immediately.
  • git log --oneline - shows a compact history of commits, one line each.
  • git diff - shows the exact line by line changes not yet staged.
  • git merge [branch name] - merges the specified branch into your current branch.
  • git restore filename — discards changes in the working directory and reverts the file back to its last committed state.
